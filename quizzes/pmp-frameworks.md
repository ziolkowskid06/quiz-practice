    answer: false

    options: ["Iterative", "Agile", "Hybrid", "Predictive"]
    answer: "Predictive"
    explanation: "Predictive is ideal when requirements are fixed, well-understood, and changes would be expensive or legally prohibited. A regulatory compliance project with a fixed scope and fixed deadline is the classic Predictive use case. There is no room for experimentation."

    options: ["Predictive", "Iterative", "Agile", "Waterfall"]
    answer: "Agile"
    explanation: "Agile is both iterative (refines through feedback) and incremental (delivers usable pieces over time). Releasing a working product immediately and adding features through regular delivery cycles is the defining characteristic of an Agile life cycle."

    answer: "Timeboxing"
    accept: ["time boxing", "timebox", "time-boxing"]
    explanation: "Timeboxing is the Agile mechanism that naturally satisfies the 'Time-bound' element of SMART goals. The Sprint is a fixed timebox — work must be completed within it. The clock does not extend, which creates a forcing function for prioritization and realistic planning."

  - id: 20
    type: multiple-choice
    question: "A sponsor says: 'I want this project delivered as soon as possible and within budget.' Which SMART element is most obviously missing?"
    options: ["Specific", "Measurable", "Achievable", "Time-bound"]
    answer: "Time-bound"
    explanation: "'As soon as possible' provides no concrete deadline. Without a fixed target date there is no Schedule Baseline to manage against, and Parkinson's Law will cause tasks to expand indefinitely. Time-bound is the element that creates the forcing function for execution."

  - id: 21

    options: ["Specific", "Measurable", "Achievable", "Time-bound"]
    answer: "Time-bound"
    explanation: "The Sprint is a fixed timebox — work must be completed within the Sprint boundary with no extension. This makes Agile inherently Time-bound by design. The Sprint Goal must be achieved within that fixed period, which also drives realistic scope selection each iteration."
