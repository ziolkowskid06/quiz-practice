      question: "XP"
      answer: "eXtreme Programming"
      accept: ["eXtreme Programming", "Extreme Programming"]
      explanation: "An Agile framework specifically for software development that emphasizes technical excellence. It introduces practices like Pair Programming, Continuous Integration, and Test-Driven Development (TDD) to improve quality and responsiveness."

    - id: 3
      type: fill-blank
      question: "ATDD"
      answer: "Acceptance Test-Driven Development"
      accept: ["Acceptance Test-Driven Development"]
      explanation: "A collaborative practice where the whole team (Product Owner, Devs, and Testers) defines acceptance criteria as automated tests before any code is written. It ensures everyone is clear on the definition of 'Done'."

    - id: 4
      type: fill-blank
      question: "DSDM"
      answer: "Dynamic System Development Method"
      accept: ["Dynamic System Development Method"]
      explanation: "An Agile framework that focuses on the full project lifecycle. It is unique because it fixes time, cost, and quality at the start and varies the scope to meet those constraints (using MoSCoW prioritization)."

    - id: 5
      type: fill-blank
      question: "LeSS"
      answer: "Large-Scale Scrum"
      accept: ["Large-Scale Scrum"]
      explanation: "A framework for scaling Scrum to multiple teams. Its core philosophy is 'More with LeSS'—meaning you should scale by removing organizational complexity rather than adding new roles and processes."

    - id: 6
      type: fill-blank
      question: "LSD"
      answer: "Lean Software Development"
      accept: ["Lean Software Development"]
      explanation: "Adapted from Toyota's Lean manufacturing. Its seven principles (like 'Eliminate Waste' and 'Defer Commitment') help Agile teams focus on delivering value quickly and efficiently."

    - id: 7
      type: fill-blank
      question: "SBE"
      answer: "Specification by Example"
      accept: ["Specification by Example"]
      explanation: "A collaborative approach to defining requirements based on realistic examples instead of abstract statements. It bridges the communication gap between business stakeholders and the technical team."

    - id: 8

      answer: "Feature Driven Development"
      accept: ["Feature Driven Development"]
      explanation: "An iterative and incremental software development process that blends several industry-recognized best practices into a cohesive whole. Its main focus is on the 'Feature' as the primary unit of delivery."

    - id: 9
      type: fill-blank
      question: "Evo"
      answer: "Evolutionary Value Delivery"
      accept: ["Evolutionary Value Delivery"]
      explanation: "A methodology that emphasizes delivering high-value requirements in short cycles. It allows for continuous feedback and ensures that the project remains aligned with business needs."

    - id: 10
      type: fill-blank
      question: "RUP"
      answer: "Rational Unified Process"
      accept: ["Rational Unified Process"]
      explanation: "An iterative software development process framework that was a precursor to many modern Agile methods, emphasizing architecture-centric development."

    - id: 11
      type: fill-blank
      question: "DA"
      answer: "Disciplined Agile"
      accept: ["Disciplined Agile", "disciplined agile"]
      explanation: "A process-decision toolkit acquired by PMI that provides guidance for choosing and tailoring an agile way of working. Unlike prescriptive frameworks such as Scrum or SAFe, DA recognizes that one size does not fit all and offers a menu of strategies to match the team\'s context."

    - id: 12
      type: fill-blank
      question: "TDD"
      answer: "Test-Driven Development"
      accept: ["Test-Driven Development", "test-driven development", "test driven development"]
      explanation: "An agile software development practice where tests are written before the production code. The short cycle is: write a failing test → write the minimum code to pass it → refactor. TDD ensures high test coverage and forces clarity about requirements before coding begins."
