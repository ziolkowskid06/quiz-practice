  - id: 1

    question: "In a Functional organizational structure, the project manager has little or no authority over budget and resources."
    answer: true
    explanation: "Correct. In a Functional structure, functional managers own resources and budgets. The PM role may be filled by a Project Expediter (zero authority) or Project Coordinator (some authority). PMI classifies Functional PM authority as 'little or none.'"

  - id: 2
    type: "true-false"
    question: "A Project Expediter has more formal authority than a Project Coordinator."
    answer: false
    explanation: "False. The Expediter has zero formal authority — they only gather and relay information. The Coordinator has some formal authority to make certain decisions. Both roles appear in Functional and Weak Matrix environments where a full PM role is not warranted."

  - id: 3
    type: "true-false"
    question: "In a Projectized organizational structure, team members are dedicated full-time to the project and report directly to the project manager."
    answer: true
    explanation: "True. Projectized structures eliminate split-resource conflicts. Team members have no competing functional manager obligations and work exclusively on the project. The tradeoff is post-project employment uncertainty when no home department exists."

  - id: 4
    type: "true-false"
    question: "A Strong Matrix organizational structure gives the project manager 'low to moderate' authority."
    answer: false
    explanation: "False. Strong Matrix gives the PM 'moderate to high' authority. 'Low to moderate' is the Balanced Matrix level. The authority ladder from lowest to highest PM control: Functional → Weak Matrix → Balanced Matrix → Strong Matrix → Projectized."

  - id: 5
    type: "true-false"
    question: "The Project Management Information System (PMIS) is classified as an Organizational Process Asset."
    answer: false
    explanation: "False. PMIS is an internal Enterprise Environmental Factor, not an OPA. This is one of the most-tested exam traps. Even though the organization provides the PMIS, it is a condition of the environment the team operates in — not a procedure or knowledge artifact the team creates."

  - id: 6
    type: "true-false"
    question: "The project team can create and update Organizational Process Assets during the course of a project."
    answer: true
    explanation: "True. OPAs are both inputs and outputs of project processes. The team creates and updates them — for example, adding entries to the Lessons Learned Repository. This updatability distinguishes OPAs from EEFs, which are inputs only."

  - id: 7
    type: "true-false"
    question: "Enterprise Environmental Factors are inputs only — the project team cannot directly modify them."
    answer: true
    explanation: "True. EEFs represent conditions the team must navigate, not assets they can change. Market conditions, culture, regulations, and stakeholder risk tolerance are all EEFs the PM plans around. Only OPAs are both inputs and outputs."

  - id: 8
    type: "true-false"
    question: "In a Balanced Matrix structure, the project manager has more authority than the functional manager."
    answer: false
    explanation: "False. In a Balanced Matrix, authority is shared equally between PM and functional manager — this equality defines it. When PM authority exceeds the functional manager's, the structure is a Strong Matrix, not Balanced."

  - id: 9

    answer: true
    explanation: "True. The Lessons Learned Register begins as a project document and transitions to an OPA when submitted to the organizational knowledge base. Future projects then use it as an input — this is the OPA lifecycle PMI tests on the exam."

Functional structures — where resources are shared and controlled by functional managers — are the most hostile to Agile adoption."
